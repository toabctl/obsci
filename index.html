Welcome to OBSCI - a CI system for the Open Build Service¶

OBSCI is a CI system for testing OBS packages. OBSCI is pretty new and there are plenty of things to do. Currently the following workflow is supported:

  • Use docker containers as Test environment

  • Defining tests in a OBS package via a _obsci config file

  • Installing RPM Test Subject in the container

Quickstart¶

  1. OBSCI can be directly installed from git into a virtual environment:

virtualenv obsci-venv
source obsci-venv/bin/activate
pip install git+https://github.com/obsci/obsci.git -e .[worker]

  1. Add a _obsci file which defines tests to your package. The file should look like:

---
tests:
  - name: first_test.sh

  1. Add a first_test.sh file to your package. This file is the actual test. If the exit code is 0, the test succeeded, otherwise it failed. first_test.sh could be written in any language. The only thing that matters is the exit code.

#!/bin/bash
echo "This is my testscript"
exit 0

  1. How the tests can be executed with the obsci-worker:

obsci-worker --obs-username tbechtold --obs-password mypassword \
    home:tbechtold openSUSE_Leap_15.1 x86_64 python-Jinja2

Here, obs-username and obs-password are needed to be able to talk to the OBS Api. home:tbechtold is the OBS project and python-Jinja2 the OBS package. This package must contain the first_test.sh and _obsci file.
